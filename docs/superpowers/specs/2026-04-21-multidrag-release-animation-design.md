# 多选拖拽堆叠卡片组释放动画设计

- 日期：2026-04-21
- 目标：让多选拖拽在插入目标列表时，堆叠卡片组中的各卡片以“同时起飞、速率分层、错落到位”的方式释放，视觉风格与现有收纳动画保持同一语言。

## 1. 背景与现状

现有多选拖拽动画已分为两个阶段：

1. `extracting`：各卡片从原列表抽离并汇入跟随指针的堆叠组。
2. `inserting`：在释放时，从堆叠组飞向目标列表中的最终插入位置。

当前插入阶段已有两项基础能力：

- 所有卡片并发启动。
- 不同层级使用不同飞行时长。

但插入阶段仍存在一个明显问题：

- 各层卡片虽然时长不同，但使用的是同一条 easing 曲线，所以视觉上更像“同一动画的不同时长版本”，而不是“同一时刻释放、但每层速率不同的成组释放”。

## 2. 用户要的效果

目标效果明确为：

- 所有卡片从堆叠组中同时开始移动。
- 越靠前的卡片越早脱离堆叠感，越快贴近目标列表。
- 越靠后的卡片起步更稳，保留更强的堆叠感，在中后段再完成收束。
- 最终形成“错落到位”，而不是串行动画，也不是整齐划一的同步平移。

这一定义排除了两种不想要的结果：

- 逐张延迟发射。
- 只增加总时长差异，不改变速率曲线。

## 3. 设计原则

### 3.1 只改插入阶段

本次只调整 `inserting` 阶段的运动学，不改：

- `extracting` 阶段的收纳动画。
- 拖拽提交顺序与最终 DOM 重排逻辑。
- 多选排序算法与目标 rect 测量逻辑。

原因：

- 收纳动画已经完成且表现稳定。
- 本次需求只针对“堆叠组释放动画”。
- 把变更限定在插入阶段，能降低行为回归风险。

### 3.2 保持“同时起飞”，反对“分批发射”

插入动画仍保持并发启动，不能改成：

- 前层先飞，后层延时起飞。
- 明显的串行或半串行节奏。

层次感应来自：

- 分层 easing。
- 分层 duration。
- 分层位移/缩放收束参数。

而不是 delay。

### 3.3 视觉语言与现有收纳动画一致

插入阶段应和收纳阶段共享以下视觉语言：

- 成组运动。
- 层次可读。
- 前后层关系清晰。
- 阴影和缩放服务于“束状收束”，不做额外花哨效果。

## 4. 方案选择

评估过三种路径：

### 方案 A：仅加大层间 duration 差

优点：
- 改动最小。

缺点：
- 只能拉开到达时机，不能形成明显的“速率有差”。
- 视觉上仍接近同一曲线的拉伸版本。

结论：不采用。

### 方案 B：分层 easing + 轻量分层 motion profile

做法：
- 保持并发启动。
- 保留层级时长差。
- 为不同层使用不同 progress 映射。
- 为不同层设置轻量不同的起始缩放/纵向收束量。

优点：
- 最直接命中需求。
- 改动聚焦，风险低。
- 能和现有 extraction 的按帧实现方式保持一致。

缺点：
- 需要调一轮参数。

结论：采用。

### 方案 C：分层路径差异做得很强

优点：
- 层次感最强。

缺点：
- 容易显得散开，而不是收束。
- 更容易破坏与现有效果的一致性。

结论：不采用。后续若方案 B 表现仍不够，再考虑小幅补强。

## 5. 最终设计

### 5.1 架构边界

只动两类代码：

1. `src/multidrag-order.ts`
   - 新增插入阶段专用纯函数：
     - 插入层级运动参数函数。
     - 插入进度映射函数。
     - 必要时把插入时长函数与通用 flight 时长函数拆开。

2. `src/main.ts`
   - 重写 `playMultiDragInsertion(...)` 中 clone 飞行动画的进度计算方式。
   - 从“统一 CSS transition 驱动”改为“按帧计算每张卡的独立 progress”。

不新增持久化设置，不新增用户配置项。

### 5.2 插入阶段的目标行为

对每张释放卡片：

- 同帧创建飞行 clone。
- 同帧开始飞行。
- 根据当前层级获得专属 motion profile。
- 根据当前层级获得专属 progress 映射。
- 在飞行过程中逐帧计算：
  - 当前位置
  - 当前尺寸
  - 当前缩放
  - 当前阴影

层级行为定义：

- 前层卡片：
  - 更快接近目标 rect。
  - 更早完成 scale 回正。
  - 更早看起来“落进目标列表”。

- 中层卡片：
  - 作为前后层过渡。

- 后层卡片：
  - 起步更稳。
  - 前段保留更多 stack 感。
  - 中后段再完成收束。

### 5.3 运动学模型

插入阶段将显式区分三类参数：

#### A. 时长

继续按层级给不同 duration，但只做轻微拉差。

预期原则：
- 前层最短。
- 后层最长。
- 总时长差存在，但不能拖沓。

#### B. progress 映射

核心变化。

输入：线性时间进度 `progress`。
输出：按层级映射后的 `easedProgress`。

要求：
- 同一时刻下，前层的 `easedProgress` 大于后层。
- 因此前层会更早脱离堆叠、更早接近目标位置。

这是本次“速率有差”的核心来源。

#### C. 分层 motion profile

每层除终点 rect 外，还定义：

- `offsetY`：起飞时保留的堆叠纵向层差。
- `startScale`：起飞时的层级缩放。
- `settleLift`：接近目标时的轻微收束量，可为 0 或极小值。

作用：
- 让插入动画保留堆叠来源感。
- 让不同层不仅“到得早晚不同”，也“飞得不完全一样”。

### 5.4 实现方式

插入阶段 clone 动画改成和 extraction 阶段同类的按帧动画：

- 不再只设置一次 `transitionDuration` 和目标 transform。
- 改为 `requestAnimationFrame` 驱动。
- 每帧按 layer-specific `easedProgress` 计算当前位置与缩放。

原因：

- 纯 CSS transition 很难稳定表达“同起飞、不同速率曲线、不同层级收束感”。
- extraction 已经是按帧实现，插入阶段采用同类策略更统一。

### 5.5 与 slot 让位动画的关系

目标列表现有 slot 让位动画继续保留，但要降存在感。

原则：
- slot 负责“让位置”。
- fly clone 负责“主视觉释放”。

因此：
- slot 的位移动画不能比 fly clone 更抢眼。
- 若当前 260ms 让位动画与新飞行动画节奏冲突，应缩短或弱化，而不是增强。

## 6. 测试设计

先写测试，再改实现。

### 6.1 `multidrag-order.test.ts`

新增或补强以下测试：

1. 插入阶段 motion profile 按层级返回不同参数。
2. 插入阶段 progress 映射在同一时间点满足：前层 > 中层 > 后层。
3. 插入阶段仍为并发启动：
   - 保留 `Promise.all(animations)` 约束。
   - 不允许出现逐张 `await this.wait(...)` 的串行释放实现。
4. 插入阶段时长仍按层级递增，且差值不过度夸张。

### 6.2 运行时行为回归保护

保留对以下关键事实的约束：

- extraction 不被改回静态目标追踪。
- preview stack 的层级构建机制不被破坏。
- 最终 DOM 提交顺序仍通过现有 `applyMultiDragFinalOrderToIds(...)` / `commitMultiDragFinalOrder(...)` 完成。

## 7. 验收标准

实现完成后，视觉验收应满足：

1. 释放时所有卡片同时开始移动。
2. 前层卡片明显更早贴向目标列表。
3. 后层卡片不是延迟起飞，而是起步更稳、收束更晚。
4. 整体像“一束卡片释放并落入目标列”，不是逐张飞出。
5. 不破坏现有收纳动画。
6. 不改变最终插入顺序。
7. 不引入明显闪烁、重影或 slot 动画抢戏问题。

## 8. 影响范围

### 修改文件

- `src/multidrag-order.ts`
- `src/main.ts`
- `multidrag-order.test.ts`

### 不修改文件

- 数据源与设置结构
- 多选排序算法本身
- 视图配置与其他拖拽场景

## 9. 风险与对策

### 风险 1：曲线差异过强，视觉变散

对策：
- 先用轻量分层参数。
- 以前层更快、后层更稳为主，不做夸张横纵向偏移。

### 风险 2：slot 让位动画抢主视觉

对策：
- 保留 slot，但让其仅服务于腾位。
- 必要时缩短 slot transition 或减弱 transform 表现。

### 风险 3：插入动画与 extraction 风格割裂

对策：
- 共享“成组运动 + 层级可读 + 收束自然”的视觉语言。
- 沿用现有阴影/缩放语汇，不引入全新夸张特效。

## 10. 不做的事

本次明确不做：

- 新增用户可调的动画设置项。
- 改 extraction 动画。
- 改多选排序规则。
- 改非多选拖拽动画。
- 引入通用动画系统重构。

## 11. 实施结论

本次需求采用“插入阶段分层 easing + 轻量分层 motion profile”的方案。

它满足：
- 同时起飞。
- 速率分层。
- 错落到位。
- 与现有收纳动画风格一致。
- 改动范围小且回归风险可控。
